> **목적**: PM/운영이 실 사용자(AGENT 1·FIELD 1·ADMIN 1)와 골든 패스 cycle을 **일관되게** 진행하고, 결과를 Phase 2 QA 게이트가 바로 채점할 수 있는 형태로 기록하기 위한 진행자(facilitator) 스크립트.
>
> **작성**: Quinn (QA Test Architect) · 2026-06-05 · Story 4.7 Phase 2 사전 정비.
> **선행 산출물**: `docs/golden-path.md`(기대결과 상세) · `docs/user-guide.md`(사용자 배포본) · `docs/backlog.md`(P3+ 트래커) · `docs/stories/4.7.story.md`(버그 로그 + 결과 표).
> **이 문서는 코드/기능을 정의하지 않는다** — 오직 *어떻게 진행하고 무엇을 관찰·기록할지*만 다룬다. 기대결과의 SoT는 `golden-path.md`다.

## 0. 진행 방식 (1 진행자 + 1 사용자)

- **진행자(facilitator)** = PM/운영. 사용자 옆에서 이 문서를 보며 **"지시문"만 소리내어 읽고**, 화면을 직접 만지지 않는다.
- **사용자(participant)** = 실 역할자. 지시문대로 **혼자** 조작한다. 막혀도 진행자는 30초간 개입하지 않고 관찰한다(= 자력 도달성 측정).
- 진행자는 각 단계의 **관찰 체크포인트**를 ✓/✗로 즉시 표시하고, 막힘·오류·불만이 나오면 **§5 즉시 버그 양식**에 그 자리에서 행을 추가한다(나중에 기억으로 채우지 않는다).
- 1 cycle = 3 역할 = 약 30~45분. 신규 P1/P2 발견 시 fix 후 **재cycle**(Deviation #6).

> ⚠️ **사이클 진행 = 외부 의존**: 본 키트는 절차만 보장한다. 사용자 3명 섭외·일정·환경 URL은 PM/운영 책임(스토리 Deviation #5).

## 2. 사전 준비 체크리스트 (cycle 시작 전, 진행자 단독 수행)

cycle 시작 **전날 또는 1시간 전**에 진행자가 단독으로 통과시킨다. 하나라도 ✗면 사용자를 부르지 말 것.

- [o] **환경 URL 확보** — 테스트 배포 환경(`docs/`의 테스트 배포 퀵스타트 참조) 또는 local dev. 데스크탑·모바일 둘 다 접속 가능.
- [o] **시드 계정 로그인 검증** — `agent1`/`field1`/`admin1` 셋 다 `dev1234`로 로그인 성공.
- [o] **카테고리 키워드 시드 확인(`local`)** — AGENT 등록 폼에서 "wifi"/"인터넷"/"자판기" 입력 시 카테고리 자동 제안이 뜬다(SW-003 fix, `LocalDataSeeder`). 운영 프로파일이면 ADMIN이 사전에 `/admin/categories`에서 키워드를 넣어야 한다.
- [o] **테스트 이슈 후보 합의** — 예: **"1층 자판기 동전 반환 안 됨"** (L1=아파트먼트v1 / L2=단말 / L3=기기미동작 / URGENT).
- [o] **모바일 디바이스 준비** — FIELD 단계용 실 스마트폰(카메라 권한 허용). 사진 3장 촬영/갤러리 준비.
- [o] **데이터 초기 상태** — 이전 cycle 잔여 이슈가 화면을 어지럽히지 않는지 확인(필요 시 시드 리셋).
- [o] **캡처 도구 준비** — 스크린샷/화면 녹화 도구. **§6 스크린샷 7장** 캡처를 cycle 중 동시에 수행.
- [o] **PII 가드 확인** — 시드 데이터(가짜 이름/전화)만 사용. 실 고객 정보 0. 캡처물에 실명·실전화 없는지 cycle 후 재확인.

## 3. 역할별 진행 스크립트

각 단계: **진행자 지시문**(읽는 말, 큰따옴표) → **관찰 체크포인트**(✓/✗) → **막히면**(개입 가이드). 기대결과 상세는 `golden-path.md` 해당 단계 참조.

### 3.1 AGENT 시나리오 — 김상담1 (`agent1`), 데스크탑

> 목표 시간: 등록 30초 / 배정 포함 1분. (front-end-spec §3.1)

**단계 A1 — 로그인 + 이슈 등록** *(golden-path 단계 1)*
- 📢 *"김상담1 계정(agent1 / dev1234)으로 로그인한 다음, 방금 전화로 접수한 '1층 자판기 동전 반환 안 됨' 건을 새 이슈로 등록해 주세요."*
- 관찰:
  - [o] 로그인 후 이슈 목록(`/issues`)에 자력 도달
  - [o] **"신규 등록" 버튼을 스스로 발견**(SW-008 fix 검증 — 헤더 우측 파란 + 버튼)
  - [o] 등록 폼에서 접수 시각·접수자가 **자동 채움**됨을 인지
  - [o] 카테고리 입력 시 **자동 제안**이 뜨고(4.2/SW-003), 적절히 선택
  - [o] 저장 후 1초 이내 상세 화면 전환을 체감
- 🆘 막히면: 30초 관찰 후에도 신규 등록 진입점을 못 찾으면 → **즉시 P1 후보**(SW-008 회귀 의심). §5 기록.

**단계 A2 — FIELD에 배정** *(golden-path 단계 2)*
- 📢 *"이 건을 현장 담당자 '이현장1'에게 배정해 주세요."*
- 관찰:
  - [o] "담당자 배정" 진입점 자력 발견
  - [o] 드롭다운에서 이현장1(`field1`) 검색·선택
  - [o] 배정 직후 상태가 `ASSIGNED`로, 활동로그에 즉시 반영
- 🆘 막히면: 드롭다운 검색이 안 되거나 본인에게 알림이 가면 → 기록.

> ⏱️ A1+A2 소요 시간 측정 → **§4 결과 표** "소요 시간"에 기록.

### 3.2 FIELD 시나리오 — 이현장1 (`field1`), **실 스마트폰**

> 목표 시간: 입력 3분 이내. (front-end-spec §3.2)

**단계 F1 — 모바일 로그인 + 본인 이슈 조회** *(golden-path 단계 3)*
- 📢 *"휴대폰 브라우저로 SMCS에 접속해서 이현장1(field1 / dev1234)로 로그인하고, 방금 배정받은 자판기 건을 찾아 상세를 열어 주세요."*
- 관찰:
  - [o] 로그인 후 모바일 홈(`/m`)으로 자동 이동
  - [x] **본인 이슈만** 카드로 표시(타 담당자 건 노출 0) - admin에서는 내이슈 표시 x
  - [o] URGENT 카드의 **빨간 막대** 시각 강조 인지
  - [o] 카드 탭 → 상세 전환이 자연스러움 / 폰트·터치 타깃 충분
- 🆘 막히면: 남의 이슈가 보이면 → **즉시 P1 후보**(권한 결함). 기록.

**단계 F2 — 사진 3장 + 조치 코멘트 + 완료** *(golden-path 단계 4)*
- 📢 *"현장에서 조치한 것처럼, 사진 3장을 첨부하고 조치 내용을 짧게 적은 뒤 '완료 처리'까지 해주세요."*
- 관찰:
  - [o] "사진 추가" → **카메라 직접 호출**(또는 갤러리 폴백) 동작
  - [x] 3장 첨부 + 업로드 진행 표시 - 사진업로드 안됨
  - [o] 조치 코멘트 입력
  - [o] **"완료 처리"가 한 번에 성공**(SW-001 fix 검증 — ASSIGNED 이슈에서 409 없이 DONE 도달)
  - [o] 완료 모달 텍스트가 명료
- 🆘 막히면: "완료 처리"에서 오류(409 등)가 나면 → **즉시 P1**(SW-001 회귀 의심). §5 기록 후 cycle 중단·dev 핸드오프.

> ⏱️ F1+F2 소요 시간 측정 → **§4 결과 표**.

### 3.3 ADMIN 시나리오 — 관리자 (`admin1`), 데스크탑

> 목표 시간: 60초 + α. (front-end-spec §3.3)

**단계 D1 — 검수** *(golden-path 단계 5)* *(선행: FIELD가 완료한 건이 있어야 함)*
- 📢 *"방금 현장에서 완료된 자판기 건을 검수해서 마무리해 주세요."* (실 운영은 AGENT 검수지만, 단일 ADMIN 진행 시 ADMIN이 대행 관찰 가능 — 또는 A 시나리오로 돌아가 agent1로 검수)
- 관찰:
  - [o] 알림 벨(4.1) 드롭다운에서 상태 변경 알림 확인
  - [o] 상세에서 **"검수 완료" / "재오픈" 두 버튼이 명확히 분리**
  - [o] (재오픈 시) 사유 입력 모달이 필수로 표시

**단계 D2 — 대시보드 + 보고서 1분 루틴** *(golden-path 단계 6)*
- 📢 *"매일 아침 출근해서 1분간 시스템을 점검한다고 생각하고, 대시보드와 보고서 보관함을 둘러봐 주세요."*
- 관찰:
  - [o] 로그인 후 `/dashboard` 자동 이동(RoleRedirect)
  - [o] KPI 4 카드 + 카테고리 차트 + 담당자별 처리량 한눈에 파악
  - [o] "보관함"(`/reports`) 진입 → PDF 미리보기 로드 속도 체감
  - [o] 벨 드롭다운의 `REPORT_READY` 알림 클릭 → `/reports` 이동
  - [-] (선택) 사용자 관리·카테고리 관리 짧게 점검
- 🆘 막히면: KPI 수치가 명백히 틀리거나 PDF가 안 열리면 → P2 후보. 기록.

> ⚠️ 보고서는 **08:00 KST cron** 산출물 — 테스트 당일 어제치 보고서가 없으면 "빈 보관함"이 정상일 수 있다. 이를 결함으로 오기록하지 말 것(golden-path 단계 6 주석).

## 4. 키보드 골든 패스 패스 (AC6) — AGENT·FIELD 각 1회 **마우스 0**

별도 패스로 진행. 사용자에게 **"이번엔 마우스·트랙패드를 절대 쓰지 말고 키보드만으로"** 안내. 4단계 중 하나라도 키보드로 도달 불가 시 P1/P2 등록.

| 단계 | 키 조작 | 통과 기준 | 결과 |
| :--- | :------ | :-------- | :--: |
| 1 로그인 | username 자동 포커스 → 입력 → Tab → password → **Enter** | 포커스 시각 표시 명확 + Enter 제출 | ☐ |
| 2 이슈 등록 | **`N`** 단축키 → 폼 Tab 순회(제목·발신자·전화·카테고리·내용) → **Ctrl+S** | N으로 진입(SW-002 fix) + Ctrl+S 저장 | ☐ |
| 3 조치 코멘트(FIELD) | 코멘트 영역까지 Tab → 입력 → "추가" 버튼 도달 → Enter | Tab 순서가 논리적 | ☐ |
| 4 완료 처리 | 상태 버튼 Tab 도달 → Enter → 확인 모달 "확인" Tab → Enter | **모달 포커스 트랩** + ESC 취소 동작 | ☐ |

- AGENT는 단계 1·2를, FIELD는 단계 3·4를 각각 키보드로 1회 통과 → **결과 표 "키보드 통과 ✓"** 컬럼 기입.
- 통과 못 한 단계는 §5에 P1(완주 불가) 또는 P2(우회 가능) 등록.

## 5. 즉시 버그 기록 양식 (발견 즉시 작성)

발견 즉시 아래 행을 채우고, cycle 종료 후 `docs/stories/4.7.story.md` **"발견 버그 로그"** 표(`UT-NNN` ID 부여)와 P3+는 `docs/backlog.md`로 이관한다.

**우선순위 즉시 판정 룰(스토리 "버그 분류 룰"):**
- **P1 (Blocker)** = 골든 패스 단계 **완주 불가** OR 데이터 손실/오염 OR 보안 결함 → **Phase 2 게이트 0건 필수**
- **P2 (Major)** = 완주는 되나 워크플로 저해 OR 우회방안 있음 OR UX 심각 저하 → **0건 필수**
- **P3 (Minor)** = 외관·엣지·미세 사용성 → 백로그 이관 OK

```
| ID(UT-NNN) | 제목 | 역할 | 시나리오 단계 | 재현 단계 | 우선순위 | 발견 시각 | 비고 |
| :--------- | :--- | :--- | :------------ | :-------- | :------- | :-------- | :--- |
| UT-001 | 사용자 전환 후 직전 사용자 이슈 목록 노출 | FIELD/ADMIN | F1 | admin1→field1 열람→admin1 재로그인 시 field1 목록 노출 | P1(보안) | cycle1 | 원인확정: logout 시 queryClient 미클리어 → 스토리 발견 버그 로그 UT-001 |
| UT-002 | 사진 업로드 실패('업로드에 실패했습니다') | FIELD | F2 | 테스트배포 docker 데스크탑 파일선택 → 실패 팝업 | P2(배포설정) | cycle1 | 원인확정: 옵션 A `prod,local` → local 상대 files.dir 가 `/app/var`(비root 불가)로 우선. fix=compose backend `SMCS_FILES_DIR=/var/smcs/files` 핀 → 스토리 UT-002 |
| UT-003 | 첨부 이미지 미리보기 불가(`/files` 프록시 누락) | FIELD/AGENT | F2 보기 | 업로드 성공 후 상세 미리보기 깨짐 | P2(프록시설정) | cycle1 | nginx/vite 가 `/files/**` 를 backend 로 안 보냄(`/api`·`/` 만) → SPA index.html 반환. fix=nginx `location /files/` + vite proxy `/files` → 스토리 UT-003 |
| UT-004 | ADMIN '내 작업'(/m) → `/me/assigned` 403 콘솔 에러 | ADMIN | 부수 | admin1 `/m` 진입 → FIELD 전용 호출 403 | P3(백로그) | cycle1 | 정상 authz(admin≠FIELD). UT-001 fix 후 표면화. AppLayout nav 가 ADMIN 에 '내 작업' 노출. 차단 아님 → backlog |
```

> P1을 발견하면 해당 역할 cycle은 중단하고 dev 핸드오프 → fix → 재cycle(Deviation #6: P1/P2 0~2건은 본 PR / 3건+는 `fix/4.7-bug-{id}` 별도 PR 시리즈).
