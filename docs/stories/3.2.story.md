# Story 3.2: 대시보드 화면

## Status: Approved

## Story

- As a **관리자**
- I want **한 화면에서 핵심 지표와 차트를 보기를**
- so that **1분 안에 현황을 파악할 수 있다**

## Acceptance Criteria (ACs)

1. 상단에 KPI 카드 4개: [선택 기간(오늘/이번주/이번달, AC5)] 신규 / [선택 기간] 처리 / 미처리 총건(현재 시점, period 무관) / [선택 기간] 평균 처리시간. (PO 정제 2026-05-23 — AC5 기간 토글 반영 명문화; openCount는 §5.9 정의에 따라 현재 스냅샷.)
2. 카테고리별 막대 차트.
3. 일자별 신규 vs 처리 추세 라인 차트.
4. 담당자별 처리량 막대 차트.
5. 기간 필터(오늘/이번주/이번달) 토글.
6. 모든 차트는 데이터 0건일 때 빈 상태 메시지 표시.

### Deviation Analysis (PO 검토 필요)

1. **[기록] KPI 카드 라벨 = 기간 반영**: AC1 문구는 "오늘 신규/오늘 처리"지만 AC5 기간 토글이 있어 선택 기간(오늘/이번주/이번달)에 따라 데이터·라벨 변동. openCount 만 현재 시점(period 무관) — 부제로 명확화.
2. **[기록] 화면 ADMIN 가드**: stats API 는 §6 인증이나, 대시보드 화면은 ADMIN 한정(§5.9 PO 비준). routes `RequireRole(['ADMIN'])`.
3. **[기록] Ant Charts lazy 격리**: 번들 가드레일 위해 dashboard 청크에서만 로드. 차트는 테스트에서 mock(jsdom canvas 미지원).
4. **[기록] 3.1 병렬 가능**: 계약 의존만 → mock 으로 선개발. 실데이터 확인은 3.1 배포 후.
