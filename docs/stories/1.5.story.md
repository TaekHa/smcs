## Story

- As a **사용자**
- I want **내 역할에 따라 로그인 후 적절한 메인 화면(빈 페이지여도)으로 안내**되기를

## Acceptance Criteria (ACs)

1. **AGENT/ADMIN** 은 로그인 후 `/issues` (빈 리스트 화면) 로 이동한다.

4. **상단 네비게이션 바**에 사용자명, 역할, 로그아웃 버튼이 표시된다.

### RoleRedirect 결정 (ADMIN 의 기본 진입)

[Source: architecture/9-frontend-architecture.md#9.5]

architecture §9.5 의 root 라우트 비고: "AGENT/ADMIN → `/issues`, FIELD → `/m`". ADMIN 은 데스크톱 진입을 기본으로 함. ADMIN 이 모바일에서 직접 작업할 일은 드물고 (사용자 환경 추정), 필요 시 `/m` 으로 직접 이동 가능 (RequireRole 이 ADMIN 허용).

### RequireRole + RequireAuth 사용 패턴

```tsx
<Route
  path="/issues"
  element={
    <RequireAuth>
      <RequireRole roles={['AGENT', 'ADMIN']}>
        <AppLayout>
          <IssueListView />
        </AppLayout>
      </RequireRole>
    </RequireAuth>
  }
/>
```

- **RequireAuth outer**: 인증 안 됨 → `/login`. 인증 됨 → 다음.
- **RequireRole inner**: 권한 미달 → `/403`. 권한 OK → 다음.
- **AppLayout 안쪽**: 네비 + 자식.

**결정**: layout 도 RequireRole 안쪽에 둠 — 403 은 layout 없는 풀스크린 결과 페이지로 표시 (UX 명확성).
