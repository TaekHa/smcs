# Story 4.3: CSV 내보내기

## Status: Review

## Story

- As a **관리자**
- I want **이슈 데이터를 CSV로 내보내기를**
- so that **엑셀 사용자가 안심하고 전환할 수 있다**

## Acceptance Criteria (ACs)

1. `GET /api/issues/export?format=csv` 가 기존 이슈 리스트 필터(`status`, `categoryL1Id`, `categoryL2Id`, `categoryL3Id`, `assigneeId`, `from`, `to`, `q`)를 받아 CSV 파일을 반환한다.
2. UTF-8 BOM 포함(엑셀 한글 호환 — 파일 첫 3바이트 `EF BB BF`).
3. 기본 컬럼(9개): `ID`, `제목`, `카테고리`(대>중>소 단일 컬럼), `우선순위`, `상태`, `담당자`, `접수일`, `처리일`, `처리 시간(분)`. **SLA 마감 컬럼은 제외** — PRD §5.6에서 SLA 정책이 v0.2에서 제거됨(Deviation #1 PO 비준 필요).
4. 발신자명/전화번호는 ADMIN 전용 옵션 컬럼으로 분리한다. `includePii=true` 쿼리 파라미터로 활성화(기본 false). 활성화 시 마지막 두 컬럼으로 `발신자명`, `발신자전화번호` 추가.
5. 결과 행 ≤ 5,000건이면 즉시 응답. 5,001건 이상이면 **400 `EXPORT_TOO_MANY_ROWS`** + "필터를 좁혀주세요"(데이터 누락 위험 차단 — Deviation #3). v2에서 비동기/스트리밍 검토.
6. 엔드포인트는 ADMIN 전용(PRD §6 명시). 미인증 401 / AGENT·FIELD 403.

### Deviation Analysis (PO 비준 완료 v0.2 — Sarah, 2026-05-26)

1. **[비준 PO v0.2] SLA 마감 컬럼 제거**: epic AC3 명시 "SLA 마감" 컬럼 vs PRD §5.6 결정(SLA 정책 v0.2 제거). **lesson 4 (architecture SoT > epic AC)** 적용 — **컬럼 제외 비준**. 이유: 빈 컬럼 출력은 엑셀 사용자 혼란 + PRD §5.6 SoT 일관성. v2 에서 SLA 재도입 시 컬럼 복원 가능. **AC3 → 9컬럼 확정**(ID/제목/카테고리/우선순위/상태/담당자/접수일/처리일/처리시간(분)).
2. **[비준 PO v0.2] 버튼 ADMIN gating**: 엔드포인트 §6 ADMIN-only 일관 — **(a) 버튼 자체 ADMIN gating 비준**. AGENT/FIELD 는 버튼 부재(보관함/대시보드 nav 패턴 일관). 이유: UX 명확성(클릭→거절 회피) + RBAC 시각화. AGENT 가 CSV 필요 시 ADMIN 요청 경로 사용.
3. **[비준 PO v0.2] 5001건+ → 400 EXPORT_TOO_MANY_ROWS**: epic AC5 "5,000건 이하 즉시 응답, 그 이상은 v2" 해석 = **(b) 400 + "필터 좁혀주세요" 비준**. 이유: truncate 는 데이터 누락을 사용자가 모르는 silent failure 위험 — 명시적 에러가 안전. v2 비동기/스트리밍 도입 시 본 가드 해제 또는 임계값 상향.
4. **[비준 PO v0.2] `phoneHashOrNull`/`startOfDay` helper 중복(surgical 복사)**: `IssueQueryService` private 메서드 — **복사 비준**. 이유: 4.3 책임 = export 만(Surgical Changes — IssueQueryService 리팩터는 범위 확장). 후속 TD 스토리에서 `IssueQueryHelpers` 공통 유틸 분리 검토 가능(TD-3 후보).
5. **[비준 PO v0.2] PII 컬럼 위치 = 후미**: includePii=true 시 `발신자명, 발신자전화번호` 마지막 2컬럼. 이유: default 9컬럼 인덱스 보존(엑셀 매크로/외부 스크립트 호환).
6. **[비준 PO v0.2] CSV escape 자체 구현**: RFC 4180 단순 규칙 → 자체 구현(Apache Commons CSV 의존성 0 추가). 단위 테스트 6케이스(평문/comma/quote/newline/quote+comma/빈문자열)로 안전성 SoT 확보. PDFBox/Ant Charts 처럼 단일 사용처 라이브러리 도입 회피 일관.
7. **[비준 PO v0.2] BOM 명시 출력**: `response.getOutputStream().write({EF,BB,BF})` 우선 + `OutputStreamWriter(UTF-8)` wrap. 이유: Java 표준 UTF-8 인코더는 BOM 미포함 → 엑셀 한글 깨짐. 통합 테스트에서 첫 3바이트 검증 의무.
