# Story 2.8: 인앱 알림 (Notification 모듈)

## Status: Approved

> 신설 배경: Epic 2 원안에 독립 스토리로 없던 **횡단 모듈**. Story 2.5 AC6(미읽음 알림 카운트)가 알림 모듈 전체를 요구하고, 2.3(#4 코멘트)·2.4(#1 배정/전이)에서 알림 생성 부수효과를 의도적으로 미뤘다. PO 결정(2026-05-22)으로 이를 단일 전용 스토리로 분리. **Epic 2 목록에 Story 2.8 추가 권장(PO/PM 후속).**

## Story

- As a **시스템 사용자(AGENT/FIELD/ADMIN)**
- I want **나와 관련된 이슈 활동(배정·코멘트·상태변경)에 대한 인앱 알림을 받고 확인하기를**
- so that **관련 변경을 놓치지 않고 협업할 수 있다**

## Acceptance Criteria (ACs)

1. 이슈 **배정/코멘트/상태변경** 시, 관련자에게 인앱 알림이 **트랜잭션 내 자동 생성**된다(kind: `ISSUE_ASSIGNED`/`ISSUE_COMMENTED`/`ISSUE_STATUS_CHANGED`).
2. 헤더 벨 아이콘에 **미읽음 카운트 뱃지**가 표시되고, **30초 주기 폴링(보이는 탭만)** 으로 갱신된다.
3. 벨(또는 카운트) 탭 시 **알림 리스트 페이지(`/notifications`)** 로 이동하며, 최신순으로 표시된다.
4. 알림 클릭 시 **`readAt` 자동 갱신** + 해당 **이슈 상세로 이동**.
5. **"모두 읽음"** 처리를 지원한다.
6. 알림은 **본인(recipient) 것만** 조회/처리 가능하다(타인 알림 접근 불가).

> 본 스토리 완료 시 **Story 2.5 AC6 충족**.

### Deviation Analysis (PO 검토 필요)

1. **[1급] 스토리 신설(off-epic) — 2.5 AC6 + 2.3/2.4 부수효과 통합**: Epic 2 원안에 없던 횡단 모듈 스토리. PO 결정(2026-05-22)으로 분리·신설. **Epic 2 목록에 Story 2.8 추가 권장**(PO/PM 후속). 2.5 AC6 충족은 본 스토리에 의존.
2. **[기록] `ISSUE_REOPENED` 알림 = Story 2.7**: 재오픈 전이(DONE→IN_PROGRESS, reason)는 2.7 → 그때 ISSUE_REOPENED 생성. 본 스토리 = 3종(ASSIGNED/COMMENTED/STATUS_CHANGED).
3. **[1급] 수신자 규칙**: 이해관계자 = {assignedTo, createdBy} − actor(중복 제거, 자기알림 금지). §7.2 "작성자 외 관련자" 의 구체화. **PO 비준 필요**(누구에게 알림 가는지 = 제품 결정).
4. **[기록] unread-count 전용 rate limit 미구현**: §6.8 120/분 전용 버킷 대신 기존 전역 300/분 의존(폴링 2/분 << 한도). 선택적 정밀화 deferred.
