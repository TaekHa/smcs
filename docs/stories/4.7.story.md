# Story 4.7: 사용자 테스트 + 버그 픽스 + 사용 가이드

## Status: Approved

> 신설 배경: Epic 4 마지막 스토리. PRD §4.7 의 5개 AC = (1) 골든 패스 시나리오 문서, (2) 실 사용자 3 역할(AGENT/FIELD/ADMIN) 시나리오 완주, (3) P1/P2 버그 100% 수정, (4) P3 이하 백로그 등록, (5) 사용 가이드 1페이지(스크린샷 포함). **테스트 + 문서 산출 중심** — 코드 변경은 발견 버그(P1/P2) fix 에 한정. 사용자 섭외/일정은 PM/운영 책임(외부 의존성, 본 스토리는 절차·산출물 정의만 보장). MVP 가동 직전 마지막 게이트.

## Story

- As a **PM(권용기) / 운영자**
- I want **실 사용자(CS 접수자 1·현장 작업자 1·관리자 1)가 골든 패스 시나리오를 완주하고, 발견된 P1/P2 버그가 모두 해결되며, 가이드 1페이지가 작성되기를**
- so that **MVP 도입 시 차단/주요 결함 없이 시스템이 가동되고, 신규 사용자가 자력으로 핵심 기능을 사용할 수 있다**

## Acceptance Criteria (ACs — PRD §4.7 + architecture §9.x)

1. **골든 패스 시나리오 문서**(`docs/golden-path.md`, 신규) — 접수 → 배정 → 모바일 조치 → 완료 → 검수 → 보고서 수신 → 알림 — 6 단계 E2E + 3 역할별 stub 시나리오(front-end-spec §3 carry-over). 각 단계는 (a) 행위자 (b) 조작 (c) 기대결과 (d) 자동 확인 가능 항목 표기.
2. **각 역할당 최소 1명 시나리오 완주** — AGENT 1 / FIELD 1 / ADMIN 1 명 = 총 3명. 본 스토리 산출물은 (a) 시나리오 문서 (b) 결과 기록 표 — 사용자 섭외/일정은 본 스토리 외 PM/운영 협의(외부 의존성, 본 스토리는 절차·기록 양식 보장).
3. **P1/P2 버그 100% 수정** — PRD 미정의이므로 본 스토리에서 **P1/P2/P3 정의 신설**(Deviation #1): **P1 (Blocker)** = 골든 패스 단계 완주 불가 OR 데이터 손실/오염 가능 OR 보안 결함 / **P2 (Major)** = 골든 패스 완주는 가능하나 워크플로 저해 OR 우회방안 있음 OR UX 심각 저하 / **P3 (Minor)** = 외관·엣지케이스·사용성 미세 결함. 본 스토리 종료 시점에 P1/P2 = 0 개 보장.
4. **P3 이하 버그 백로그 등록** — `docs/backlog.md`(신규) 또는 본 스토리 내 버그 테이블에 P3+ 항목 기록(제목/재현 단계/우선순위/접수일). v2 트리거 = 본격 issue tracker 도입.
5. **사용 가이드 1페이지** `docs/user-guide.md`(신규) — 3 역할별 핵심 동작 5~7 단계 + 스크린샷 PNG. 사용자가 자력 도달 가능한 단순 톤. PII 마스킹된 스크린샷만 사용(실명/전화번호 노출 금지).
6. **키보드만으로 골든 패스 통과** (architecture §9.x / ARCHITECTURE.md L1008 mandated, PO 정정) — 마우스 사용 없이 Tab/Shift+Tab/Enter/Space/방향키/단축키(N 신규 등록·Ctrl+S 저장)만으로 **로그인 → 이슈 등록 → (FIELD 시점) 조치 코멘트 → 완료 처리** 4단계 완주 가능. AGENT/FIELD 두 역할 시나리오에 각 1회 포함. 스크린리더 검증은 MVP 필수 아님(v2). 통과 시 본 스토리 § "사용자 테스트 결과" 표의 **키보드 통과** 컬럼 ✓.
