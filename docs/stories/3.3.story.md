# Story 3.3: 일간/주간 PDF 보고서 생성

## Status: Done

## Story

- As a **시스템**
- I want **일/주간 보고서를 PDF로 생성하기를**
- so that **관리자가 다운로드하거나 (3.5 보관함에서) 미리볼 수 있다**

## Acceptance Criteria (ACs)

1. `GET /api/reports/daily?date=YYYY-MM-DD` 가 해당 일자(KST) PDF를 반환한다.
2. `GET /api/reports/weekly?week=YYYY-Www` 가 해당 주(ISO 주차, KST) PDF를 반환한다.
3. PDF는 회사 로고 영역(MVP는 텍스트만), 표지, 요약 KPI, 카테고리 분포, TOP 5 카테고리, 담당자별 처리량, 미처리 리스트(컬럼: ID / 제목 / 우선순위 / 담당자, 기본 정렬 priority DESC → created ASC §5.2)를 포함한다.
4. A4 1-2페이지 분량. 미처리 리스트가 1페이지 초과 시 최대 30건까지 표시하고 "이하 N건 생략 — 보관함 PDF 참조" 풋노트를 표기한다.
5. 한글 폰트가 정상 임베드된다(나눔고딕 등).
6. PDF 생성에 단위 테스트가 작성되어 있다.

**잔여 관찰(비차단):**
1. **운영 데이터 증가 시 메모리 위험** — `loadOpenIssues`가 SQL `LIMIT` 없이 전체 미처리를 받음. MVP 데이터 소량 가정에는 OK이나 운영 누적 시 N+1 메모리 폭증 가능. 3.4 스케줄러도 동일. **권장(v2/3.4)**: `Pageable.ofSize(OPEN_LIST_MAX + 1)` 적용 + footnote 카운트는 `issueRepository.countByStatusIn(...)` 별도 호출로 정확성 유지.
2. **`ReportPeriod.forWeek` 가독성** — `LocalDate.now(KST)`를 with 베이스로 사용. 결과는 `with(MONDAY)` 후 day-of-week 의존 제거로 시간 비의존 보장되지만, 코드 가독성 noise. **대안**: `LocalDate.of(year,1,4).with(IsoFields.WEEK_OF_WEEK_BASED_YEAR, week)`로 명시. 현 동작 정확하므로 비차단.
3. **`ReportController` 로컬 핸들러 격리** — `IllegalArgumentException`을 컨트롤러 로컬에서만 400 변환. 다른 컨트롤러가 같은 예외 던지면 일관성 없음 — 현재 다른 호출처 없어 OK. 향후 같은 패턴 확산 시 전역 변환 또는 전용 예외 클래스(`InvalidReportPeriodException`) 도입 검토.
