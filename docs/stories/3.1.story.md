# Story 3.1: 통계 집계 백엔드

## Status: Draft

## Story

- As a **백엔드(시스템)**
- I want **이슈 통계를 효율적으로 집계할 수 있는 API를 제공하기를**
- so that **대시보드(3.2)와 보고서(3.3)가 동일한 데이터를 사용한다**

## Acceptance Criteria (ACs)

1. `GET /api/stats/dashboard?period=today|week|month` 이 신규/처리/미처리/평균처리시간을 반환한다.
2. 카테고리별 건수, 담당자별 처리량, 우선순위별 분포를 제공한다.
3. 시간대별(일자 또는 주차 단위) 추세 데이터를 제공한다.
4. 응답 시간 P95 < 500ms.
5. 집계 로직에 단위 테스트가 작성되어 있다.

### Deviation Analysis (PO 검토 필요)

1. **[기록] V5 = resolved_at 인덱스만**: 나머지 집계 인덱스 V2 기존. 첫 신규 Flyway(스키마 무변경 종료, PO 비준). reports/notifications 변경은 3.3/3.4 별도 마이그레이션(V6/V7).
2. **[기록] 집계 = JdbcTemplate native SQL**: KST date 그룹(`AT TIME ZONE`)은 JPQL 불가(Architect/§5.9). LocalDataSeeder 선례. 계산/범위 로직은 분리해 순수 단위테스트.
3. **[기록] period 스코프 정의**: openCount=period 무관(현재 open); newCount/byCategory/byPriority=period 내 created; resolvedCount/byAssignee/avg=period 내 resolved; trend=일자별 new vs resolved. (3.2 대시보드가 이 의미로 렌더.)
4. **[기록] 권한 = §6 isAuthenticated**: stats API 는 인증(전 역할 호출 가능). 대시보드 화면 ADMIN 제한은 3.2 라우트 가드. §6 PO 비준.
5. **[기록] AC4 P95<500ms = 인덱스 설계 보장**: 엄밀 부하측정은 MVP 범위 밖. 인덱스 적중 근거를 코드/Dev Notes 로. week/month/today 정의: today=오늘 KST, week=이번 ISO 주(월~일) KST, month=이번 캘린더 월 KST.
