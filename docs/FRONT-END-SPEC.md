## 0. 한 줄 요약

> **"엑셀처럼 빠르게 입력하고, 카카오톡처럼 직관적으로 협업하며, 한 번의 클릭으로 인사이트를 얻는 내부 도구."**
> 화려함보다 **속도·정확성·신뢰**를 우선한다.

## 1. UX 원칙 (Sally의 10가지 원칙)

| # | 원칙 | 구체적 적용 |
|---|------|------------|
| 1 | **속도가 곧 UX다** | CS 등록 폼은 단축키 + 자동 저장. 모든 리스트는 우선순위 정렬 |
| 2 | **3중 시각 강조** | 우선순위/상태는 색상 + 텍스트 + 아이콘 (색약 친화) |
| 3 | **사용자 위치를 잊지 않게 한다** | 빵부스러기, 활성 메뉴 강조, 상태 진행 바 |
| 4 | **모바일은 모바일답게** | 카드 스택, 큰 터치, 한 손 도달 영역 고려 |
| 5 | **에러는 친절하게** | 에러 메시지는 "왜" + "어떻게 해결" 둘 다 |
| 6 | **로딩 상태도 디자인이다** | 스피너 X, Skeleton O |
| 7 | **빈 상태는 기회다** | "데이터 없음"이 아니라 "다음에 할 일" 제안 |
| 8 | **알림은 방해하지 않는다** | 조용한 뱃지 + 부드러운 펄스 (소리 X) |
| 9 | **접근성은 처음부터** | 키보드만으로 모든 흐름 통과 가능 |
| 10 | **신뢰는 디테일에서** | 마이크로 카피, 적절한 여백, 정렬 일관성 |

## 2. Personas

### 2.1 페르소나 #1 — 박지영 (32세, CS 접수자)

| 항목 | 내용 |
|------|------|
| **직무** | CS팀 주임. 1년 6개월 경력. |
| **하루** | 9:00-18:00, 점심 1시간. 통화 30~50건/일 |
| **환경** | 헤드셋 통화. 24인치 듀얼 모니터. 키보드 위주. |
| **습관** | 단축키 사랑함. 동시에 여러 일 처리. 통화 끊으면 즉시 정리해서 잊지 않음. |
| **목표** | "통화 중에는 듣는 데 집중하고, 끊자마자 30초 안에 카드 정리하기" |
| **고민** | "지금은 엑셀에 적는데 행 추가 위치 헷갈리고, 전화번호 중복 체크가 안 됨" |
| **시스템에 바라는 것** | 빠른 입력, 자동 채움, 과거 이력 즉시 조회, 단축키 |
| **싫어하는 것** | 모달 여러 개 떠서 흐름 끊김, 페이지 이동 시 입력값 날아감 |

## 3. User Journey Maps

### 3.1 박지영 (CS 접수자) — 통화 1건 처리

```
[전화 벨] → [헤드셋 받음, 동시에 시스템 N 단축키]
   ↓
[등록 폼 열림 — 제목 입력란에 자동 포커스]
   ↓
[고객: "안녕하세요, 1234호인데요..." → 듣는 동안 메모 키워드만 빠르게]
   ↓
[전화번호 입력 시 → 과거 이력 1건 토스트로 표시 "이 번호 지난주 동일 이슈 있음"]
   ↓
[카테고리 트리 모달 — L1 → L2 → L3 자동 진행]
   ↓
[Ctrl+S로 저장 → 상세 페이지 이동 (1초 미만)]
   ↓
[필요시 담당자 배정 드롭다운 → 김민호 선택]
   ↓
[자동 알림 발송 (인앱) → 통화 종료]

⏱️ 목표 시간: 등록 30초 / 배정 포함 1분
```

**감정 곡선:**
- 😐 시작 (긴장) → 😊 자동 채움 발견 (안심) → 😄 단축키로 빠름 (만족) → 😌 끝

### 3.2 김민호 (현장 작업자) — 한 건 출동/조치/보고

```
[차량에서 폰 잠금 해제]
   ↓
[SMCS PWA 아이콘 탭 → 자동 로그인 유지 → 모바일 홈]
   ↓
[카드 스택 위에서 두 번째 카드 = URGENT (빨간 막대)]
   ↓
[카드 탭 → 상세: 1층 자판기 미동작, 1234호 김O옥님]
   ↓
[현장 도착 → 시스템 안 봄. 작업 30분 진행]
   ↓
[작업 완료 후 차로 복귀 → 폰 다시 꺼냄]
   ↓
[상세 페이지 "사진 추가" → 카메라 직접 호출 → 3장 촬영]
   ↓
[조치 내용 입력 (짧게): "전원 보드 교체, 정상 동작 확인"]
   ↓
["완료 처리" 버튼 → 확인 모달 → 완료]
   ↓
[자동으로 다음 카드로 이동]

⏱️ 목표 시간: 입력 3분 이내
```

**감정 곡선:**
- 😟 도착 전 (모르겠음) → 😊 정보 다 있음 (확신) → 😄 사진 빨리 첨부 → 😎 다음 일로

### 3.3 이수진 (관리자) — 매일 아침 1분 루틴

```
[09:00 출근. 노트북 켜기]
   ↓
[브라우저 즐겨찾기 → SMCS]
   ↓
[로그인 후 자동으로 대시보드]
   ↓
[상단 4개 KPI 카드: 어제 신규 12 / 처리 8 / 미처리 누적 5 / 평균 4.2시간]
   ↓
[차트: 카테고리별 분포 — "voip/pbx > 단말" 평소보다 ↑ 발견]
   ↓
[좌측 메뉴 "보고서" → 어제 일간 보고서 열기 (별도 페이지 iframe)]
   ↓
[PDF 1페이지 빠르게 스캔]
   ↓
[필요시 특정 이슈 클릭해서 상세 확인]

⏱️ 목표 시간: 60초
```

## 4. Information Architecture (사이트맵)

```
SMCS
├── /login                            (Public)
│
├── 📋 CS / Admin 영역 (데스크톱 우선)
│   ├── /issues                       이슈 리스트 (메인)
│   ├── /issues/new                   신규 등록
│   ├── /issues/:id                   이슈 상세
│   ├── /dashboard                    대시보드 (Admin)
│   ├── /reports                      보고서 보관함 (Admin)
│   │   ├── ?tab=daily
│   │   └── ?tab=weekly
│   └── /reports/:kind/:date          PDF 미리보기 페이지
│
├── 📱 Field 영역 (모바일 우선)
│   ├── /m                            내 작업 카드 스택
│   ├── /m/issues/:id                 이슈 상세 + 사진 + 조치
│   └── /m/notifications              알림 페이지
│
├── 🔔 공통
│   └── /notifications                알림 페이지 (데스크톱)
│
└── ⚙️ Admin 전용
    ├── /admin/users                  사용자 관리
    └── /admin/categories             카테고리 관리 (L1/L2/L3 탭)
```

### 4.1 네비게이션 구조

**데스크톱 헤더:**
```

```

## 5. Visual Design System

### 5.1 색상 팔레트 (Ant Design 5 기본 + 시맨틱)

| 토큰 | Hex | 사용처 |
|------|-----|--------|
| **Primary** | `#1677ff` | 주 액션 버튼, 링크, 활성 상태 |
| **Success** | `#52c41a` | 완료/검수 상태, 성공 토스트 |
| **Warning** | `#fa8c16` | HIGH 우선순위, 경고 |
| **Danger** | `#ff4d4f` | URGENT 우선순위, 삭제, 에러 |
| **Info** | `#13c2c2` | NEW 상태 |
| **Purple** | `#722ed1` | VERIFIED 상태 |
| **Gray-9** | `#262626` | 본문 텍스트 |
| **Gray-6** | `#8c8c8c` | 보조 텍스트, LOW 우선순위 |
| **Gray-4** | `#d9d9d9` | 보더 |
| **Gray-2** | `#fafafa` | 배경 (테이블 헤더) |

**우선순위 색상 매핑 (PRD FR16 일치):**
```
URGENT  ━━━━━  #ff4d4f  🔥  긴급
HIGH    ━━━━━  #fa8c16  ⚠️  높음
NORMAL  ━━━━━  #1677ff  •   보통
LOW     ━━━━━  #8c8c8c  ○   낮음
```

**상태 색상:**
```
NEW         #13c2c2  접수
ASSIGNED    #1677ff  배정
IN_PROGRESS #fa8c16  진행중
DONE        #52c41a  완료
VERIFIED    #722ed1  검수
```

### 5.2 타이포그래피

| 요소 | 폰트 | 크기 | 굵기 | 행간 |
|------|------|------|------|------|
| 페이지 제목 | Pretendard / Noto Sans KR | 24px | 600 | 1.4 |
| 섹션 제목 | 동상 | 18px | 600 | 1.5 |
| 본문 | 동상 | 14px | 400 | 1.6 |
| 보조 | 동상 | 12px | 400 | 1.5 |
| 모바일 카드 제목 | 동상 | 16px | 500 | 1.4 |
| 단축키 표시 | monospace | 12px | 500 | - |

> Ant Design 기본 폰트 스택을 따른다. 한국어는 Pretendard 또는 Noto Sans KR.

### 5.3 스페이싱 (8pt grid)

`4, 8, 16, 24, 32, 48, 64` — 모든 여백/간격은 이 값들 중 하나.

### 5.4 보더 / 라운드

- 라운드: **6px** (Ant Design 기본)
- 카드 보더: `1px solid #f0f0f0`
- 그림자: `0 2px 8px rgba(0,0,0,0.08)` (호버), `0 4px 16px rgba(0,0,0,0.12)` (모달)

### 5.5 아이콘

- Ant Design Icons (개별 import — 트리 셰이킹)
- 우선순위 이모지는 텍스트 폴백
- 활동 로그 아이콘:
  - 📝 코멘트 / 🎯 배정 / 🔄 상태 변경 / 📷 첨부 / ✅ 완료 / 🔁 재오픈

## 6. Wireframes — 핵심 화면 10개

> 표기법: ASCII 박스 + 주석. 실제 픽셀 단위가 아닌 **상대 비율/구조**만 표현.

### 6.1 화면 1: 로그인 (`/login`)

```
                    ┌─────────────────────────────┐
                    │                             │
                    │          [ SMCS ]           │  ← 로고 (텍스트 또는 심플 아이콘)
                    │      CS 이슈 관리 시스템    │
                    │                             │
                    │  ┌─────────────────────┐    │
                    │  │ 사용자명             │    │  ← username (autoFocus)
                    │  └─────────────────────┘    │
                    │                             │
                    │  ┌─────────────────────┐    │
                    │  │ 비밀번호  ●●●●●●  👁 │    │
                    │  └─────────────────────┘    │
                    │                             │
                    │  ┌─────────────────────┐    │
                    │  │      로그인          │    │  ← Primary button, Enter 가능
                    │  └─────────────────────┘    │
                    │                             │
                    │  [⚠ 5회 실패 시 10분 잠금]   │  ← 보조 안내
                    │                             │
                    └─────────────────────────────┘
```

**인터랙션:**
- 페이지 로드 시 username 필드 자동 포커스
- Enter 키로 로그인 제출
- 실패 시 폼 위에 빨간 알럿 + 남은 시도 횟수 표시
- 잠금 시 카운트다운 타이머 ("9분 23초 후 재시도 가능")

**상태:**
- 로딩: 버튼 텍스트 → "로그인 중..." + 스피너
- 에러: 표준 에러 메시지 (사용자 친화 카피)
- 잠금: 폼 비활성화 + 카운트다운

### 8.2 마이크로 인터랙션 (Q5 = b)

**새 알림 도착 펄스:**
```css
@keyframes notification-pulse {
  0%   { transform: scale(1);   }
  50%  { transform: scale(1.18); }
  100% { transform: scale(1);   }
}

.bell-icon.has-new {
  animation: notification-pulse 0.6s ease-in-out 1;
}
```
- 카운트가 증가할 때마다 1회 재생
- `prefers-reduced-motion: reduce` 사용자에게는 비활성화

**상태 전이 (Steps 진행 바):**
- 다음 단계로 이동 시 새 단계 점이 부드럽게 채워짐 (300ms)

**카드 호버 (데스크톱):**
- `transform: translateY(-2px)` + 그림자 강화 (150ms)

**버튼 클릭 피드백:**
- Ant Design 기본 ripple 유지

### 8.3 자동 저장 (Draft)

**등록 폼 sessionStorage Draft:**
- 사용자 입력 5초마다 저장
- 페이지 이탈 시 (`beforeunload`) 변경사항 있으면 경고
- 다시 방문 시 "💾 작성 중이던 내용이 있습니다. 복원하시겠어요?" 토스트
- 저장 성공 시 Draft 자동 삭제

## 10. Accessibility Spec (WCAG 2.1 Level A 화면별)

| 화면 | 체크 항목 |
|------|----------|
| **로그인** | username 자동 포커스 / Enter로 제출 / 비밀번호 보기 토글 / 에러 메시지 `role="alert"` |
| **이슈 리스트** | 테이블 헤더 `<th scope="col">` / 정렬 변경 시 `aria-sort` / 행 선택 키보드 가능 / 우선순위 색상 + 텍스트 라벨 |
| **등록 폼** | 모든 input `<label>` 명시 / 필수 표시 `aria-required="true"` / 에러 메시지 `aria-describedby` / 카테고리 모달 진입 시 자동 포커스, 닫힐 때 트리거 버튼으로 복귀 |
| **이슈 상세** | 진행 바 `role="progressbar"` + `aria-valuenow` / 활동 로그 `role="log"` `aria-live="polite"` (실시간 추가 시) |
| **모바일 카드** | `role="button"` + `tabIndex=0` + Enter/Space 키 처리 / 우선순위 `aria-label="긴급 우선순위"` |
| **모바일 상세** | 사진 추가 버튼 `aria-label="사진 추가"` / textarea `<label>` 명시 / 완료 버튼 비활성 상태 `aria-disabled="true"` + 이유 안내 |
| **대시보드** | 차트는 표 형태 데이터 대체 제공 (`<details>` 안에 `<table>`) / KPI 화살표는 텍스트 라벨 병기 ("증가/감소") |
| **알림 드롭다운** | `<Dropdown>` 키보드 네비게이션 / `role="menuitem"` / 새 알림 도착 시 `role="status"` 라이브 영역 |
| **모달** | 진입 시 자동 포커스 / Esc 닫기 / 닫힐 때 트리거로 복귀 / focus trap |

**색상 대비 검증:**
- 모든 텍스트 vs 배경 4.5:1 이상 (Level AA 권장. MVP Level A는 3:1)
- Ant Design 기본 토큰은 대부분 충족 — 단, gray-6 (`#8c8c8c`) 텍스트는 흰 배경에서 4:1로 경계선. **본문에는 gray-9 사용 권고**

## 11. Responsive Breakpoints

| 디바이스 | 폭 | 적용 |
|---------|-----|------|
| 모바일 (소) | 320 ~ 575 | 단일 컬럼, 카드 스택 |
| 모바일 (대) / 태블릿 (세로) | 576 ~ 767 | 동상, 일부 2컬럼 |
| 태블릿 (가로) | 768 ~ 991 | 데스크톱 레이아웃 적용 시작 |
| 데스크톱 | 992 ~ 1199 | 표준 |
| 데스크톱 (대) | 1200 ~ 1599 | 표준 + 여백 |
| 데스크톱 (초대) | 1600+ | 최대 폭 1440px로 제한 + 가운데 정렬 |

**모바일 ↔ 데스크톱 전환:**
- 같은 코드베이스, Ant Design 반응형 grid (`<Row gutter>`, `<Col xs sm md lg xl>`)
- `/m/*` 경로는 **모바일 전용 화면** — 데스크톱에서도 동작은 하지만 컨텐츠 폭 480px로 제한

## 12. Animation Guidelines

| 변화 | Duration | Easing |
|------|----------|--------|
| 모달 열기/닫기 | 200ms | `ease-in-out` |
| 드롭다운 열기 | 150ms | `ease-out` |
| 카드 호버 | 150ms | `ease-out` |
| 페이지 전환 | 250ms 페이드 | `ease-in-out` |
| 상태 진행 바 채우기 | 300ms | `ease-in-out` |
| 알림 펄스 | 600ms 1회 | `ease-in-out` |
| Skeleton shimmer | 1500ms 반복 | `linear` |

**원칙:**
- 사용자가 의도한 동작에만 애니메이션 (자발성)
- 300ms를 넘는 애니메이션은 가능한 피함
- `prefers-reduced-motion: reduce` 사용자에게는 모든 애니메이션 즉시 완료 (CSS 0ms 오버라이드)
