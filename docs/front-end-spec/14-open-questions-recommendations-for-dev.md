## 14.2 Dev에게 권하는 구현 순서

1. **Story 1.1~1.5 (Foundation)**: 디자인 시스템 토큰 먼저 설정 (color, spacing). Ant Design ConfigProvider로 일관성 확보
2. **Story 2.1 등록 폼** → 가장 사용자 가치 큰 화면. 단축키와 자동 저장은 처음부터 포함
3. **Story 2.2 리스트** → 정렬/필터/검색이 잘 동작하면 나머지 화면이 따라옴
4. **Story 2.6 모바일 사진** → 실제 폰에서 자주 테스트. 데스크톱 시뮬레이터로는 불충분
5. **Story 3.x 대시보드/보고서** → 시각 디자인 가장 신경 쓸 부분. 차트 라이브러리 학습 시간 확보
