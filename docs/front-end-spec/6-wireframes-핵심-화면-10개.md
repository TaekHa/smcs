# 6. Wireframes — 핵심 화면 10개

> 표기법: ASCII 박스 + 주석. 실제 픽셀 단위가 아닌 **상대 비율/구조**만 표현.

## 6.1 화면 1: 로그인 (`/login`)

```
                    ┌─────────────────────────────┐
                    │                             │
                    │          [ SMCS ]           │  ← 로고 (텍스트 또는 심플 아이콘)
                    │      CS 이슈 관리 시스템    │
                    │                             │
                    │  ┌─────────────────────┐    │
                    │  │ 사용자명             │    │  ← username (autoFocus)
                    │  └─────────────────────┘    │
                    │                             │
                    │  ┌─────────────────────┐    │
                    │  │ 비밀번호  ●●●●●●  👁 │    │
                    │  └─────────────────────┘    │
                    │                             │
                    │  ┌─────────────────────┐    │
                    │  │      로그인          │    │  ← Primary button, Enter 가능
                    │  └─────────────────────┘    │
                    │                             │
                    │  [⚠ 5회 실패 시 10분 잠금]   │  ← 보조 안내
                    │                             │
                    └─────────────────────────────┘
```

**인터랙션:**
- 페이지 로드 시 username 필드 자동 포커스
- Enter 키로 로그인 제출
- 실패 시 폼 위에 빨간 알럿 + 남은 시도 횟수 표시
- 잠금 시 카운트다운 타이머 ("9분 23초 후 재시도 가능")

**상태:**
- 로딩: 버튼 텍스트 → "로그인 중..." + 스피너
- 에러: 표준 에러 메시지 (사용자 친화 카피)
- 잠금: 폼 비활성화 + 카운트다운

**핵심 디자인 결정:**
- 우선순위 정렬이 **기본** (URGENT 위로). 시각적으로 빨간 막대가 가장 위
- 카테고리 컬럼은 `L1>L2`만 표시 (L3는 호버 툴팁)
- 행 클릭 시 상세 페이지 이동
- 우측 상단 "+신규 등록" 버튼 + 단축키 `N` 힌트 노출

**인터랙션:**
- 검색은 디바운스 300ms
- 필터 변경 시 URL 쿼리 파라미터 동기화 (북마크 가능)
- 컬럼 헤더 클릭 → 정렬 변경. 새로고침 시 기본값 복귀
- 우선순위 색상 = 좌측 4px 막대 + 뱃지

```
│                                                                                 │
└─────────────────────────────────────────────────────────────────────────────────┘
```

**핵심 디자인 결정:**
- **단일 컬럼 위주 + 일부 2컬럼** — 위에서 아래로 자연스러운 입력 흐름
- 제목 필드에 자동 포커스 (페이지 로드 시)
- 전화번호 입력 자동 포맷팅 (`01012345678` → `010-1234-5678`)
- 전화번호 매칭 시 **인라인 토스트** (모달 X — 흐름 끊지 않음)
- 카테고리는 **단일 입력 필드처럼** 보이지만 클릭하면 트리 모달
- 자동 입력 (텍스트 변경 시) 카테고리 키워드 매칭되면 빈 카테고리 자동 채움
- **자동 저장 (Draft)**: 통화 중 입력하다 새로고침 등으로 잃지 않도록 5초마다 sessionStorage 저장
- 저장 성공 시 즉시 상세 페이지 이동 (페이지 전환 < 1초)

**인터랙션:**
- `Ctrl+S` / `Cmd+S` → 저장
- `Esc` → 취소 확인 모달 (변경사항 있을 때만)
- `Alt+1~4` → 우선순위 (긴급/높음/보통/낮음)
- 필수 누락 시: 첫 번째 누락 필드로 자동 스크롤 + 빨간 보더 + 메시지

## 6.4 화면 4: 카테고리 트리 선택 모달 (Q3 = c)

```
                ┌──────────────────────────────────────────────┐
                │  카테고리 선택                          ✕ ESC│
                ├──────────────────────────────────────────────┤
                │                                              │
                │  대분류 (L1)        중분류 (L2)    소분류 (L3)│
                │  ┌─────────────┐   ┌─────────┐   ┌──────────┐│
                │  │ ◯ 아파트v1   │   │◯ 관리자웹│   │◯ 기기미동작││
                │  │ ◉ 아파트v2   │ → │◉ 단말   │ → │◉ 기기오동작││
                │  │ ◯ voip/pbx   │   │◯ 서버   │   │◯ 로그인오류││
                │  │              │   │◯ 입주민앱│   │          ││
                │  └─────────────┘   └─────────┘   └──────────┘│
                │                                              │
                │  📍 선택: 아파트먼트v2 > 단말 > 기기오동작    │
                │                                              │
                │                          [취소]  [확인 Enter]│
                └──────────────────────────────────────────────┘
```

**핵심 디자인 결정:**
- 3열 가로 배치 (데스크톱) / 3행 세로 (모바일)
- **자유 조합** — 한 컬럼 선택해도 다른 컬럼 자동 필터링 X (PRD 가정)
- 하단 빵부스러기로 선택 결과 실시간 표시
- 키보드: Tab으로 컬럼 이동, ↑↓로 선택, Enter로 확정
- 자주 쓰는 조합 5개를 모달 상단에 "최근 사용" 칩으로 표시 (선택)

**모바일 변형:**
```
┌──────────────────────────────┐
│ 카테고리 선택          ✕ ESC │

│ └──────────────────────────┘ │
│                              │
│ 소분류 (L3)                  │
│ ┌──────────────────────────┐ │
│ │ ◯ 미동작 ◉ 오동작 ◯ 로그인 │ │
│ └──────────────────────────┘ │
│                              │
│ 📍 아파트v2 > 단말 > 오동작  │
```

- 권한 없는 사용자는 [변경], [상태 변경] 버튼 비활성화 (배경색 회색)

**[상태 변경 ▾] 드롭다운:**
- 현재 상태에서 가능한 다음 상태만 표시
- 예: 진행중일 때 → ["완료로 변경", "재오픈 (사유 필요)"]
